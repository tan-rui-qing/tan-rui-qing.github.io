## 🌟 项目介绍
本系统是一款模拟算法竞赛全流程的交互式工具，涵盖**选手训练、笔试选拔、多日正赛、奖牌评定**等核心环节，还原真实算法竞赛的压力、能力成长与排名机制，适合算法竞赛爱好者体验竞赛全流程。

## 🚀 快速开始
### 环境要求
- 现代浏览器（Chrome/Firefox/Edge/Safari 最新版）
- 无需额外安装依赖，直接运行 HTML 文件即可

### 启动步骤
1. 下载项目所有文件（HTML/CSS/JS 等）至本地
2. 双击打开 `index.html` 文件（或在浏览器中拖拽打开）
3. 等待页面加载完成后，即可进入系统开始操作

## 📋 核心功能与操作流程
### 1. 初始化阶段
- 进入系统后，首先设置**参赛选手数量**（支持 1~20 人）
- 为每位选手命名，可选是否设置队长
- 点击「开始训练」进入正式流程

### 2. 训练阶段（20 周）
- 系统自动模拟 20 周算法训练，每周更新选手能力值（DP/数据结构/数学/字符串/图论/思维/代码）
- 训练过程中会随机触发「天赋激活」「压力波动」，影响选手能力成长
- 训练结束后展示所有选手的能力面板，可查看各维度成长数据

### 3. 笔试选拔阶段
- 笔试为基础筛选环节，满分 100 分，难度低于正赛
- 系统自动模拟笔试答题过程，根据选手能力值计算得分
- 笔试得分计入最终总分（占比 20%）

### 4. 正赛阶段（默认 5 天，可配置）
#### 每日流程
- 每天包含 30 轮答题环节，每轮针对 1 道题目进行解答
- 每道题目包含 DP/数据结构/数学/字符串/图论 5 个维度难度，以及思维难度、代码难度
- 答题过程中会触发「卡题」「心态崩溃」「负面天赋」等随机事件，影响答题进度
- 每道题支持部分分（10/30/60/100 分），根据解题进度自动判定得分

#### 题目难度规则
- Day1-Day2：难度较低（选手中位数能力 × 0.9），大概率完成 4/5 道题
- Day3-Day5：难度较高（选手中位数能力 × 1.25），仅能完成 1/3 道题
- 题目难度随天数/题号递增，浮动范围逐步扩大

### 5. 最终排名与奖牌评定
- 正赛结束后，系统计算选手「笔试得分 + 正赛总分」的累计分数
- 按累计分数降序生成最终排名
- 奖牌评定规则：
  - 金牌：前 10% 选手
  - 银牌：前 20% 选手（含金牌，新增 10%-20%）
  - 铜牌：前 30% 选手（含金/银牌，新增 20%-30%）
- 展示详细排名表、每日每题得分明细，以及冠军感言

## 📖 关键概念解释
### 1. 能力值体系
| 维度       | 说明                     | 取值范围 |
|------------|--------------------------|----------|
| DP         | 动态规划解题能力         | 0~100    |
| 数据结构   | 各类数据结构应用能力     | 0~100    |
| 数学       | 数论/组合数学等解题能力  | 0~100    |
| 字符串     | 字符串匹配/处理能力      | 0~100    |
| 图论       | 图算法解题能力           | 0~100    |
| 思维难度   | 题目逻辑分析难度         | 0~100    |
| 代码难度   | 代码实现复杂度           | 0~100    |

### 2. 压力值
- 取值范围 0~100，越高代表选手压力越大
- 压力值过高会触发「心态崩溃」，暂停本轮答题
- 训练/比赛过程中会随机波动，队长选手压力值增长更快

### 3. 总分计算
- 最终总分 = 笔试得分（20% 权重） + 正赛每日得分总和
- 正赛每日得分 = 当日所有题目得分之和
- 所有分数上限为「比赛总分上限」（笔试 100 + 每日满分 × 正赛天数）

## ❓ 常见问题
### Q1: 为什么部分题目难度显示为个位数？
A: 系统已优化难度保底机制：Day1-Day2 单维度难度 ≥25，Day3-Day5 ≥35，若仍出现异常，可检查选手初始能力值是否过低（系统已默认保底选手综合能力 ≥30）。

### Q2: 分数线为什么会超过满分？
A: 系统强制限制分数线 ≤ 比赛总分上限，且虚拟选手分数也会限制在满分内，若仍异常可刷新页面重新运行。

### Q3: 操作无响应/页面加载失败？
A: 
1. 确认浏览器为最新版本
2. 清除浏览器缓存后重新打开
3. 检查文件是否完整（HTML/CSS/JS 缺一不可）

### Q4: 如何调整正赛天数/每日题目数？
A: 可修改 JS 代码中 `gameState.dailyTotalDays`（正赛天数）、`gameState.dailyProbCount`（每日题目数）参数，默认值分别为 5、5。

## ⚠️ 注意事项
1. 系统为模拟工具，所有能力值、分数、随机事件均为算法生成，不代表真实竞赛水平
2. 建议在大屏设备（电脑）上运行，移动端可能存在布局适配问题
3. 若需保存比赛数据，可截图或复制页面文本至本地
4. 禁止修改核心逻辑参数（如奖牌比例、难度系数）后用于商业用途
